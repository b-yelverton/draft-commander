<task type="auto">
  <name>Task 2: Add @supports glass effects for .hdr, .modal-overlay, .room-top, .section-hdr</name>
  <files>index.html</files>
  <action>
Add a single `@supports` block after the theme transition rules (after line 342, the .no-transitions rule). This block contains glass overrides for all four target elements.

**Insert after line 342 (after the `.no-transitions` rule, before the `/* Empty states */` comment):**

```css
/* Glassmorphism — progressive enhancement */
@supports (backdrop-filter:blur(1px)){
  .hdr{background:rgba(var(--surface-rgb),var(--glass-opacity));-webkit-backdrop-filter:blur(var(--glass-blur));backdrop-filter:blur(var(--glass-blur))}
  .modal-overlay{background:rgba(var(--shadow-rgb),.5);-webkit-backdrop-filter:blur(8px);backdrop-filter:blur(8px)}
  .room-top{background:rgba(var(--surface-rgb),var(--glass-opacity));-webkit-backdrop-filter:blur(var(--glass-blur));backdrop-filter:blur(var(--glass-blur))}
  .section-hdr{background:rgba(var(--surface-rgb),var(--glass-opacity));-webkit-backdrop-filter:blur(8px);backdrop-filter:blur(8px)}
}
```

Key details for each element:

**`.hdr` (sticky header, z-index:100):**
- Overrides solid `background:var(--surface)` with `rgba(var(--surface-rgb),var(--glass-opacity))`
- Uses `var(--glass-blur)` (12px) for consistent blur token
- Both `-webkit-backdrop-filter` and unprefixed for Safari < 18

**`.modal-overlay` (fixed overlay, z-index:200):**
- Already has `backdrop-filter:blur(2px)` on line 234 — the @supports block overrides with stronger `blur(8px)`
- Increases background from `.4` to `.5` opacity for better depth separation
- Uses hardcoded `8px` not `var(--glass-blur)` — modal overlay blur is independent of the glass system token
- Do NOT add `-webkit-backdrop-filter` prefix separately from the unprefixed — they go together in the same rule

**`.room-top` (pick clock bar):**
- Overrides solid `background:var(--surface)` from line 141
- Uses glass tokens for consistency
- The clockPulse animation on `.room-top.on-clock` uses `box-shadow` and `border-bottom-color`, both of which work independently of backdrop-filter — no conflict

**`.section-hdr` (sticky section header, z-index:5):**
- Overrides solid `background:var(--surface)` from line 160
- Uses hardcoded `8px` blur (slightly less than header's 12px for visual hierarchy)
- Content (player rows) scrolls beneath this in .board-panel, so the blur IS visible

**Elements intentionally NOT given glass:**
- `.pos-filters` — sits directly below `.section-hdr`; double glass creates visual noise
- `.p-row` — 100+ rows, GPU cost too high
- `.modal` (content box) — double blur with `.modal-overlay` looks over-processed; keep solid `background:var(--surface)` for text readability

**IMPORTANT:** The existing `.modal-overlay` rule on line 234 keeps its current `backdrop-filter:blur(2px)` as the base. The `@supports` block overrides it when supported. This means unsupported browsers still get `blur(2px)` — that is fine; the base rule already has `backdrop-filter` without an `@supports` guard, and in unsupported browsers it simply has no effect while the background color remains visible.

Actually, there is a correctness concern: line 234 has `backdrop-filter:blur(2px)` WITHOUT an `@supports` guard. This is technically fine (unsupported browsers ignore the property), but for consistency, the existing `backdrop-filter:blur(2px)` on line 234 should be removed and the full modal-overlay glass should live exclusively in the `@supports` block. However, since the base `background:rgba(var(--shadow-rgb),.4)` on line 234 is already semi-transparent (not solid), it works correctly as a fallback — unsupported browsers show a semi-transparent dark overlay without blur. Do NOT change line 234. Only add the override in the `@supports` block.
