  <action>
**Add `scheduleRoomUpdate()` and `updateRoomData()` functions** in the Rendering section (near `render()`):

```javascript
let _roomUpdatePending=false;
function scheduleRoomUpdate(){
  if(_roomUpdatePending)return;
  _roomUpdatePending=true;
  requestAnimationFrame(()=>{
    _roomUpdatePending=false;
    if(S.view==='room')updateRoomData();
    else render();
  });
}

function updateRoomData(){
  const teams=S.draft?.settings?.teams||12;
  const ba=getBestAvailable(S.posFilter==='ALL'?null:S.posFilter);
  const recs=getRecommendations();
  const recent=[...S.picks].reverse().slice(0,15);

  const bp=document.querySelector('.board-panel');
  if(bp){const st=bp.scrollTop;bp.innerHTML=renderBoardInner(ba,recs);bp.scrollTop=st;}

  const sb=document.querySelector('.sidebar');
  if(sb){const st=sb.scrollTop;sb.innerHTML=renderSidebarInner(recs,recent);sb.scrollTop=st;}

  bindEvents();
  updatePills();
}
```

Key details:
- `_roomUpdatePending` is a module-level boolean gate that prevents multiple rAF callbacks from stacking
- If the user has navigated away from room view (e.g., back to setup), fall back to full `render()`
- `bindEvents()` is safe to call after partial updates because it uses `.onclick=` assignment (idempotent, not addEventListener)
- Scroll position is captured before innerHTML replacement and restored after

**Rewire source loaders** -- In each of these functions, replace `render()` with `scheduleRoomUpdate()`:

1. **`loadKTC()`** (end of function, line ~474): Change `updatePills();render();` to `updatePills();scheduleRoomUpdate();`
2. **`parseKTCPaste()`** (end of function, line ~546): Change `updatePills();render();` to `updatePills();scheduleRoomUpdate();`
3. **`loadDD()`** (end of function, line ~581): Change `updatePills();render();` to `updatePills();scheduleRoomUpdate();`
4. **`loadDP()`** (end of function, after the try/catch around line ~620): Change `updatePills();render();` to `updatePills();scheduleRoomUpdate();`

Note: `updatePills()` is called BEFORE `scheduleRoomUpdate()` in each case. This is intentional -- pills update synchronously and immediately (they target `#srcPills` in the header which is outside `#app`), while the room content update is batched via rAF.

Do NOT change the `render()` call in `pollDraft()` (line ~862) -- polling already correctly does a full render when new picks arrive, which is a different code path from source loading.

Do NOT change `loadFC()` -- FantasyCalc is loaded synchronously before the room view renders, so it uses the normal render path.
