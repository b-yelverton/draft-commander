  <action>
**1. Draft selection grid (line 225):**
Change the `minmax(320px, 1fr)` to use `min()` function so it never exceeds available width:
```css
/* FROM: */ .draft-grid{display:grid;grid-template-columns:repeat(auto-fill,minmax(320px,1fr));gap:10px;width:100%;max-width:1000px}
/* TO: */   .draft-grid{display:grid;grid-template-columns:repeat(auto-fill,minmax(min(320px,100%),1fr));gap:10px;width:100%;max-width:1000px}
```
The `min(320px, 100%)` ensures the column minimum never exceeds the container width, preventing horizontal overflow on viewports under 320px + padding.

**2. Setup screen mobile padding:**
Add to the existing `@media(max-width:599px)` block (created by Plan 02-01) or create a new one if Plan 02-01's block is not present yet:
```css
@media(max-width:599px){
  .setup{padding:var(--sp-xl) var(--sp-md)}
  .draft-select{padding:var(--sp-xl) var(--sp-md)}
}
```
Reduces horizontal padding on setup and draft-select screens from `var(--sp-xl)` (20px) to `var(--sp-md)` (12px) on phones, giving more width for content.

Note: If Plan 02-01 already created a `@media(max-width:599px)` block, add these rules inside that same block rather than creating a duplicate. If this plan executes before or in parallel with 02-01, create the block and the executor of the other plan should merge them.

**3. Loading screen verification:**
Read the `.loading` CSS rule (line 60). It uses flexbox centering with no fixed widths -- confirm it is already responsive. No changes needed. Add a code comment if desired: `/* .loading — already responsive, no fixed widths */`

**4. Tip CTA on narrow viewports:**
The `.tip-cta` (line 251) has `max-width:440px` and `width:100%` with flex layout. On very narrow phones, the tip button might crowd the text. Add to the phone media query:
```css
@media(max-width:599px){
  .tip-cta{flex-direction:column;text-align:center}
  .tip-cta .tip-btn{align-self:center}
}
```
This stacks the tip CTA text and button vertically on phones.
