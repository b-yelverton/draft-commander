<objective>
Convert the room view from desktop-first to mobile-first CSS layout.

Purpose: The room grid currently uses `1fr 340px` columns with `overflow:hidden` and fixed height, which clips content and prevents scrolling on phones and tablets. This plan restructures the base styles for mobile (single column, natural document flow) and uses `min-width` breakpoints to restore the desktop layout at 1000px+.

Output: Room view usable on phone (< 600px) and tablet (600px-1000px) viewports with sidebar stacked below board, horizontally scrollable board rows, and responsive room-top bar.
</objective>

<task type="auto">
  <name>Task 1: Convert room grid and sidebar to mobile-first layout</name>
  <files>index.html</files>
  <action>
Modify the CSS in the `<style>` block to convert the room layout from desktop-first to mobile-first. All changes are CSS-only.

**1. Room grid base style (line 66):**
Replace the current `.room` rule:
```css
/* FROM: */ .room{display:grid;grid-template-columns:1fr 340px;grid-template-rows:auto 1fr;gap:0;height:calc(100vh - 52px);overflow:hidden}
/* TO: */   .room{display:grid;grid-template-columns:1fr;grid-template-rows:auto auto 1fr;gap:0;min-height:calc(100vh - 52px)}
```
Key changes: single column (mobile base), `auto auto 1fr` rows (room-top, board, sidebar), `min-height` instead of `height`, removed `overflow:hidden` so page scrolls naturally on mobile.

**2. Existing max-width media query (line 67):**
Replace the `@media(max-width:1000px)` rule with a `@media(min-width:1000px)` rule that restores desktop layout:
```css
/* FROM: */ @media(max-width:1000px){.room{grid-template-columns:1fr;grid-template-rows:auto auto 1fr}}
/* TO: */   @media(min-width:1000px){.room{grid-template-columns:1fr 340px;grid-template-rows:auto 1fr;height:calc(100vh - 52px);overflow:hidden}}
```
This restores the two-column grid, fixed height, and overflow:hidden only on desktop.

**3. Board panel (line 84):**
Add `overflow-x:auto` to the `.board-panel` rule so wide player rows scroll horizontally on narrow viewports:
```css
/* FROM: */ .board-panel{overflow-y:auto;border-right:1px solid var(--border);background:var(--surface)}
/* TO: */   .board-panel{overflow-y:auto;overflow-x:auto;border-right:none;background:var(--surface)}
```
Remove `border-right` from base (not needed on mobile). Add it back in the desktop media query.

Add to the `@media(min-width:1000px)` block:
```css
.board-panel{overflow-x:hidden;border-right:1px solid var(--border)}
```

**4. Sidebar (line 122):**
Change sidebar base style to swap `border-left` for `border-top`:
```css
/* FROM: */ .sidebar{overflow-y:auto;background:var(--surface);display:flex;flex-direction:column;border-left:1px solid var(--border)}
/* TO: */   .sidebar{overflow-y:auto;background:var(--surface);display:flex;flex-direction:column;border-top:2px solid var(--border)}
```
Add to the `@media(min-width:1000px)` block:
```css
.sidebar{border-top:none;border-left:1px solid var(--border)}
```

**5. Header mobile padding (line 26):**
Add a mobile padding reduction. Add BEFORE the `@media(min-width:1000px)` block (or in a new `@media(max-width:599px)` block near line 67):
```css
@media(max-width:599px){
  .hdr{padding:10px 12px;gap:10px}
  .hdr-logo{font-size:18px}
}
```
This reduces header padding and logo size on phones to prevent overflow.

**6. Room-top mobile stacking (near line 68):**
Add mobile styles for `.room-top` to prevent overflow:
```css
@media(max-width:599px){
  .room-top{padding:var(--sp-sm) var(--sp-md);gap:var(--sp-sm);flex-direction:column;align-items:stretch}
  .room-top .draft-meta{margin-left:0;flex-wrap:wrap;justify-content:flex-start}
  .room-tabs{overflow-x:auto;-webkit-overflow-scrolling:touch}
}
```
On phones: room-top stacks children vertically, draft-meta wraps, tabs can scroll horizontally.

Group the new mobile-specific rules together in a single `@media(max-width:599px)` block placed right after the `@media(min-width:1000px)` block for the room grid.
