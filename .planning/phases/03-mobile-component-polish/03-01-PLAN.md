  <action>
Modify the existing `@media(max-width:599px)` block on line 68 of index.html. The block is currently a single compressed line. Expand it to multi-line format for maintainability, then add the new rules.

**MOBI-03: Header collapse -- add these rules inside the media query:**
```css
.src-pills{display:none}
.tip-btn{display:none}
.hdr-sub{display:none}
.hdr-right{gap:8px}
```
This hides source pills, tip button, and "Live Draft Assistant" subtitle on mobile. The header will show: logo + Switch Draft button + live indicator. `updatePills()` still works since `display:none` does not remove elements from DOM.

**MOBI-04: Tab scrolling enhancement -- modify the existing `.room-tabs` rule and add `.room-tab` rules:**

The existing `.room-tabs{overflow-x:auto;-webkit-overflow-scrolling:touch}` rule in the media query needs to be merged with new properties:
```css
.room-tabs{overflow-x:auto;-webkit-overflow-scrolling:touch;scroll-snap-type:x mandatory;scrollbar-width:none;-ms-overflow-style:none}
.room-tabs::-webkit-scrollbar{display:none}
.room-tab{flex-shrink:0;scroll-snap-align:start}
```
- `scroll-snap-type:x mandatory` makes tabs snap to edges when swiping
- `scrollbar-width:none` and `::-webkit-scrollbar{display:none}` hide the scrollbar (touch scrolling is sufficient)
- `flex-shrink:0` prevents tabs from compressing -- they maintain natural width and overflow into the scroll container
- `scroll-snap-align:start` sets each tab as a snap point

Do NOT create a duplicate `@media(max-width:599px)` block. All rules go inside the existing one. Preserve all existing Phase 2 rules in the block.

<task type="auto">
  <name>Task 2: Touch target sizing for all interactive elements</name>
  <files>index.html</files>
  <action>
Add touch target sizing rules inside the same `@media(max-width:599px)` block modified in Task 1. All interactive elements that currently fall below 44px must be enlarged.

**Add these rules to the media query block:**

```css
/* Touch target baseline for buttons */
.room-tab,.pos-filter,.refresh-btn,.btn,.btn-sm{min-height:44px;display:inline-flex;align-items:center;justify-content:center}

/* Room tabs: increase vertical padding to fill 44px naturally */
.room-tab{padding:10px 14px}

/* Position filters: increase vertical padding */
.pos-filter{padding:10px var(--sp-md)}

/* Player rows: ensure 44px minimum */
.p-row{min-height:44px}

/* Trade chips and future pick chips */
.trade-chip,.fpick-chip{min-height:44px;display:inline-flex;align-items:center}

/* Trade inputs */
.trade-input input{min-height:44px}

/* Range sliders: expand touch area (visual track stays thin via browser defaults) */
.weight-row input[type=range]{height:44px}

/* Prevent 300ms tap delay on all interactive elements */
button,a,.p-row,.trade-chip,.fpick-chip{touch-action:manipulation}
```

**Key considerations:**
- `.room-tab` already has `flex-shrink:0;scroll-snap-align:start` from Task 1. The new rules here ADD `min-height:44px` and padding. CSS will merge these naturally since they are non-conflicting properties. If you write them as a single combined `.room-tab` rule, that is also fine.
- `min-height:44px` requires `display:inline-flex` (or flex) with `align-items:center` to vertically center content. Without this, text hugs the top of the 44px box.
- `.tip-btn` and `.src-pills` are already hidden on mobile (Task 1), so they do NOT need touch target fixes.
- `.sb-rec` (sidebar cards ~60px+), `.opp-card` (~60px+), and `.setup-row input` (~44px) already meet the 44px threshold -- do NOT add unnecessary rules for them.
- `.log-row` is display-only, not interactive -- do NOT add touch target rules.

After adding all rules, consolidate any `.room-tab` rules that appear multiple times in the media query into a single rule if possible, to avoid redundancy.
