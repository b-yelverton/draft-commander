Two targeted fixes in index.html:

**Fix 1 -- CSS specificity for .tip-btn on mobile (line 72):**
The rule `.tip-btn{display:none}` at line 72 inside `@media(max-width:599px)` has equal specificity to `.tip-btn{display:inline-flex;...}` at line 274, and the base rule wins because it appears later in source order.

Change line 72 from:
  `.tip-btn{display:none}`
to:
  `.hdr-right .tip-btn{display:none}`

This increases the specificity (0,2,0 vs 0,1,0) so the mobile hide rule wins regardless of source order. The tip button lives inside `.hdr-right` in the DOM (confirmed by the HTML structure at line ~394).

Do NOT move the media query block or reorder base styles -- specificity increase is the minimal, safe fix.

**Fix 2 -- Switch Draft button visibility in bindEvents() (lines 1904-1908):**
Currently the JS shows switchDraftBtn only when `S.view==='room'` and hides it on all other views. This causes the mobile header to look sparse/broken on draftSelect screen.

Replace the switchDraftBtn logic block (lines 1904-1908) with:
```javascript
const sdBtn=document.getElementById('switchDraftBtn');
if(sdBtn){
  if(S.draftList.length>0){
    sdBtn.style.display='';
    if(S.view==='room'&&S.draftList.length>1){
      sdBtn.onclick=()=>{stopPolling();S.view='draftSelect';S.draft=null;S.picks=[];render();};
    } else if(S.view==='room'){
      sdBtn.onclick=()=>{stopPolling();S.view='setup';render();};
    } else {
      sdBtn.onclick=()=>{S.view='setup';S.draftList=[];render();};
    }
  } else {
    sdBtn.style.display='none';
  }
}
```

This shows the Switch Draft button whenever the user has connected to at least one draft (S.draftList.length > 0), keeping the header consistent across room and draftSelect views. The onclick behavior adapts per view:
- room + multiple drafts: go to draftSelect
- room + single draft: go to setup
- draftSelect/other: go to setup and clear draft list
